# SlimeEasy 设计说明

## 配置与语言边界

- `config.yml` 只保存功能开关、研究等级和运行参数，不保存面向玩家的文案。
- `lang/<language>.yml` 按功能和组件层级保存物品、分类、研究、菜单、消息与展示名称，不使用扁平数字键。
- 物品和 UI 图标统一使用包含 `name` 与 `lore` 的基础节点；Lore 采用 YAML `|-` 多行块。
- Kotlin 代码只引用稳定的语义基础键；动态值通过具名 `{placeholder}` 注入，颜色码统一使用 `&`。
- `SEText.localized` 与 `GuiItems.localized` 分别封装 Slimefun 物品和 Adventure UI 图标构造。
- 自定义语言缺失文件或键时回退到插件内置 `zh_CN.yml`，避免升级后出现空文本。
- `/se reload` 同时重载配置与语言。后续读取的菜单和消息更新；Slimefun 注册阶段冻结的物品、分类、研究及已缓存界面文本需重启生效。

## 依赖边界

- DecentHolograms 只以 `compileOnly` 和 Paper 可选依赖接入；公共护目镜服务通过项目自有接口与反射工厂隔离，
  未安装 DH 时不得加载任何 `eu.decentsoftware.holograms` 类型。
- 护目镜直接构造默认不可见的临时 Hologram，再单独授权佩戴者；禁止调用内部会执行 `showAll()` 的创建捷径。
- Slimefun 能源只使用公共属性 API。实时流量定义为相邻采样的电量净变化，不反射能源网络局部调度数据，
  也不把净变化描述成可分别计量的实际输入与输出。
- 护目镜筛选写入物品自身 PDC，五个字段分别保存禁用的 ItemGroup、机器种类、物品 ID、附属名与工作状态；
  禁用集合协议保证旧物品及未来新增注册项默认可见，多个维度以 AND 语义共同生效。
- 筛选界面复用项目 ChestMenu，分类取 Slimefun ItemGroup 注册表；功能类型按多方块、发电、模板机、配方展示、
  能源组件的优先级互斥归类。工作状态只读取公开 MachineProcessHolder 操作或发电输出，无法验证时归入未知。
- 护目镜把多方块中心按与区块对齐的 `16×16×16` 单元建立全服共享空间索引；首次访问扫描单元，后续玩家复用，
  方块变化按结构一格影响半径局部失效，区块装卸清除边界相邻列，4096 单元上限按最近访问顺序淘汰；
  每轮默认最多建立 4 个新单元（配置至少为 1），并轮换在线玩家遍历起点，避免首次进入区域的集中扫描与多玩家饥饿。
- 普通 Slimefun 方块在每轮刷新内按区块共享缓存快照；工作状态与能源详情按目标共享。每位玩家仍独占 DH 句柄，
  但只在文本真实变化时更新发包，从而保持私有可见性同时降低重复读取与网络流量。
- 工程师夜视护目镜使用 Slimefun 原生 `SlimefunArmorPiece` 施加与夜视眼镜一致的夜视效果，不另建药水任务；
  基础与组合型号共用工程师研究和扫描/筛选身份判断，配方仍要求真实 `NIGHT_VISION_GOGGLES` 作为输入。

项目直接使用 Paper、Adventure 与 Slimefun API，不依赖 TabooLib。需要原版方块实体、真实交互和假玩家语义的路径通过 `paperweight-userdev` 与 Paper dev bundle 编译为 Mojang 映射的强类型 NMS/CraftBukkit 调用，不使用运行时反射；因此升级 Paper 时必须重新解析 dev bundle 并检查二进制签名。发行过程不使用 Shadow，纯 Java `SlimeEasyLoader` 在任何 Kotlin 类加载前通过 Paper 默认 Maven 镜像解析与编译器同版本的 Kotlin 标准库；Loader 常量与版本目录必须同步，首次离线启动前需已有服务端依赖缓存。语言加载由 `I18n` 独立完成，避免把框架配置能力引入业务模块。

## 磁盘存储边界

- 磁盘管理器使用雕纹书架的六个原生书槽作为安装状态与方块外观的唯一真相源，独立 ChestMenu 只展示镜像并执行受校验的安装 / 拆卸。
- 磁盘物品 PDC 只保存不可变 UUID 与格式版本；库存明细以 UUID 为主键写入 Slimefun UniversalData，由服务器当前配置的 SQLite、MySQL 或 PostgreSQL 后端持久化。
- 管理器仅在内存中聚合六张磁盘，不在方块 BlockData 保存第二份内容；翻页箱继续使用 `StorageCacheUtils` 方块数据，同样由 Slimefun SQL 后端持久化。
- 物品分类沿用 `ItemKey` 的完整 ItemStack meta/NBT 相等性。容量按八分之一字节整数单位计算，严格实现 `种类数 × x × 8 + 物品数 / 8`，并单盘限制 64 种。
- 存储网络通过 `CargoBufferBlock` 的可覆盖持久化钩子访问管理器；管理器不注册隐藏货运菜单，避免其 0/1 槽覆盖雕纹书架真实书槽。

## 网络端口 IO 边界

- 控制器 ticker 是端口路由的唯一调度者；端口不注册独立 ticker，避免一个网络被按端口数量重复扫描。
- 端口缓冲保存已从来源真实扣除的物品，玩家、货运和主动 IO 不共享镜像库存；主动 IO 直接在相邻目标与网络间事务传输。
- `CARGO / ACTIVE / BOTH` 只控制货运访问和主动扫描，玩家缓冲与缓冲到网络的桥接始终可用；缺少模式数据的旧端口按 `CARGO` 解释。
- 主动 IO 访问普通容器、网络外部的 `CargoBufferBlock` 及实现 `InventoryBlock` 的 Slimefun 标准菜单；输入端只读 `WITHDRAW` 输出槽，输出端按物品写入动态 `INSERT` 输入槽。当前网络的存储成员和端口、经验存储仍跳过，并以端口 owner 校验 `INTERACT_BLOCK`。
- 过滤名单约束全部物品流；六面配置只约束主动相邻 IO。目标拒收时优先回退来源或网络，最终无法回退的数量才在端口位置掉落。

### 安全与限制

- UniversalData 查询同时校验 UUID 对应记录的 Slimefun ID 与磁盘规格，阻止篡改 UUID 后覆盖其它通用数据。
- 非空磁盘在容量策略入口被拒绝，避免递归存储和数据规模指数膨胀；空磁盘允许作为普通物品存储。
- Slimefun 权限服务控制管理器 UI 访问；已打开界面在每次热插拔前重新校验方块身份，避免方块破坏后向原坐标写入。
- 被永久销毁或通过管理指令复制的 UUID 无法可靠感知：前者可能留下孤立 UniversalData，后者会让副本指向同一逻辑磁盘。这是“物品仅携带轻量 UUID”模型的已知限制。

## 领地保护边界

- 领地以核心、最多35面旗帜和完整世界 UUID/Int 区块坐标组成；核心与每面旗帜都声明以中心区块为原点的3×3覆盖，同领地覆盖允许重叠，权限区块是真实覆盖并集，任何增删后都以四方向 BFS 保证核心可达。
- `territories.yml` 是版本化动态存档，使用临时文件原子替换。版本4把核心也升级为3×3覆盖并从版本1-3重建区块并集；加载时校验主人唯一、跨领地不重叠、同世界、35面旗帜上限、覆盖并集和旗帜映射，仍可解析的损坏记录保持锁定。
- `TerritoryProtectionModule` 把六类行为权限接入 Slimefun `ProtectionManager`。本模块允许不代表最终允许，WorldGuard 等其它保护模块仍必须同时通过。
- 原版事件与 Slimefun 共用同一权限模型；投射物追溯真实玩家来源，屠夫的固定假玩家伤害通过短生命周期击杀标记还原为机器放置者再逐目标鉴权，避免丢失领地主人或成员身份。机器 owner 使用独立版本标记确认绑定：缺少标记的历史数据即使 owner 非空也视为旧机器，领地外兼容放行；机器被领地覆盖后首次保护查询会覆盖历史值并绑定当时的领地主人。已有版本标记但 UUID 损坏则失败关闭。爆炸、实体改块和跨边界活塞/液体/火焰不允许绕过玩家权限。
- 行为权限、管理权限和领地主人彼此分离。管理权限细分成员、行为权限、区块、旗帜样式和设置；管理者不能授予自身或默认成员管理能力。新成员加入时复制默认模板，之后修改模板不会回写现有成员。
- 旗帜编辑器直接读取玩家背包中被点击旗帜的底色与全部图案层，并取消该次点击以保证模板不移动、不消耗；持久化只保存白名单化的底色和图案注册表键，不保存可能携带名称、Lore、PDC 的完整 ItemStack。同步会替换对应颜色的旗帜材质、复制墙旗朝向或立旗旋转并写入 Banner TileState 图案，按坐标保存的 Slimefun 身份保持不变，新放置旗帜立即继承模板；破坏仍由 Slimefun 返回默认无花纹领地旗帜。
- Minecraft 26.2/26.3 的 `BannerBlockEntity` 把花纹保存在 `DataComponents.BANNER_PATTERNS` 对应的 `BannerPatternLayers` 中；同步以干净 BannerMeta 模板生成 NMS ItemStack，强类型调用原版 `applyComponentsFromItemStack` 与 `setChanged`，再用 `ServerChunkCache.blockChanged` 触发包含方块实体数据的区块广播，Bukkit TileState 仅作为 NMS 写入失败时的兼容回退。
- 成员邀请只展示操作者可见且尚未加入的在线玩家，并使用 Slimefun `PlayerHead` 生成头像；选择头像后还需点击确认按钮，打开列表、选择和确认三个阶段都重新校验领地、成员管理权及在线状态。旧邀请在接受时再次校验发起者权限与领地主人，防止撤权或转让后继续生效。转让聊天输入仅接受在线成员精确名称且 60 秒超时，不拼接执行服务器命令；可点击接受命令只携带 Minecraft 玩家名。普通成员只读查看自身权限，成员移除需要二次确认。
- 边界变化在最终未取消的移动或传送事件后通过动作栏提示；同一领地内部不提示，禁入消息按玩家限频。`slimeeasy.territory.admin` 只绕过本领地模块；真实机器放置者为 OP 时，机器前置与屠夫伤害阶段直接放行，不借用内部假玩家的固定 OP 身份给普通放置者提权。
- 手持领地旗帜时，每10 tick仅向持有者按每0.5格一个粒子的密度绘制附近可管理领地的覆盖并集外沿；同领地重叠区域不重复绘制，5区块显示半径限制粒子与网络开销。

## 变更历史

### 2026-07-12 - 新增工程师护目镜

**决策**: 新增铁头盔槽位诊断工具，使用 DecentHolograms 2.10.1 为每位佩戴者维护独立临时全息图，
扫描附近已加载的 Slimefun 方块和注册多方块，并展示公共 API 可验证的能源信息。

**变更理由**: 提供不打开菜单即可观察机器身份、电量、额定耗电、发电量和净流量的工程视图，同时避免实体命中箱、
跨玩家可见性与强制外部依赖。

**影响范围**: 实用工具注册、可选依赖、空间索引与共享扫描任务、DH 生命周期、能源采样、物品 PDC 筛选、
ChestMenu、潜行快捷交互、配置、语言和关闭清理。

**显示模式协议**: 护目镜 PDC 保存“附近显示”或“瞄准显示”，旧物品、未知值与重置操作统一回退附近显示。
瞄准模式复用本轮空间索引与共享目标结果，再以 `getTargetBlockExact(radius, FluidCollisionMode.NEVER)` 过滤视线目标，
不额外扫描世界；普通机器只接受自身方块，多方块接受匹配结构中任意非 `null` 成员，避免把通配空位当作结构命中。

**全息图高度**: 临时全息图启用 DecentHolograms `down-origin`，先由 DH 按每条 `HologramLine` 的真实高度向上排列，
再读取实际末行坐标并平移整个页面，使最后一行精确落在机器顶部目标点；能源行增减或不同内容行高都无需手工估算位移。
无法通过公共 API 判断状态的目标仍归入“状态未知”筛选，但省略无诊断价值的状态文本行。

**筛选界面**: 五类页签分别使用独立材质和名称颜色，当前页叠加隐藏说明文字的附魔光效；中央模式按钮同时展示筛选汇总，
点击后直接切换显示模式。固定槽位与 PDC 键均属于持久布局协议，修改时必须同步语言和旧物品回退行为。

### 2026-07-12 - 修复自动机器领地身份

**变更内容**: 屠夫伤害事件从固定假玩家还原真实机器放置者后再检查领地权限，让 OP 放置者在机器前置与伤害阶段直接放行，并以独立版本标记为旧机器增加领地主人 UUID 惰性绑定；自动点击器同时接入左右键独立保护检查。

**变更理由**: 防止领地主人或已有实体攻击权限的成员因假玩家 UUID 不属于领地而被错误阻止，同时避免普通玩家借内部假玩家的 OP 视图绕过保护。

**影响范围**: 自动机器 owner 数据、自动点击器、屠夫伤害来源归属、领地原版事件监听与使用文档。

### 2026-07-12 - 在线玩家头颅邀请菜单

**变更内容**: 成员邀请改为 Slimefun PlayerHead 在线玩家分页列表，选择目标后通过独立确认按钮发送邀请，并在邀请接受时补充撤权与转让竞态校验。

**变更理由**: 避免玩家手工输入名称，同时防止菜单快照和旧邀请绕过最新在线状态或成员管理权限。

**影响范围**: 领地成员菜单、邀请会话、聊天输入监听、本地化文本与 README。

### 2026-07-12 - 迁移到无 Shadow Loader 发布

**变更内容**: 移除 Shadow 插件与 fat Jar，改由纯 Java Paper PluginLoader 从默认 Maven 镜像加载 Kotlin 标准库，CI 上传并发布普通 Jar。

**变更理由**: 减少插件产物体积并复用 Paper 的依赖缓存与类路径管理，同时保证 Loader 自身不依赖尚未加载的 Kotlin。

**影响范围**: Gradle 构建、插件 Loader、GitHub Actions、下载链接与运行依赖说明。

### 2026-07-12 - NMS 调用改为强类型 userdev

**变更内容**: 接入 Paper `paperweight-userdev` 与 dev bundle，把旗帜组件同步、自动点击和机器假玩家的运行时反射全部替换为 Mojang 映射强类型调用；假玩家使用专用 ServerPlayer/CraftPlayer 包装维持内存 OP 视图，不修改 `ops.json`。

**变更理由**: 消除反射签名解析与私有字段访问，并让旗帜花纹沿原版方块实体组件及区块广播链可靠同步。

**影响范围**: Gradle 依赖、旗帜同步、自动点击器、屠夫机器假玩家、日志文本与版本升级约束。

### 2026-07-12 - 核心默认覆盖升级为3×3

**变更内容**: 领地核心从单区块升级为中心3×3初始覆盖，存档升级到版本4并迁移版本1-3，同时把边界粒子间距加密到0.5格。

**变更理由**: 让新建领地无需额外旗帜即可获得完整基础区域，并提高边界预览的连续性。

**影响范围**: 核心创建校验、区块索引、拓扑重建、存档迁移、物品文案和粒子显示。

**变更理由**: 提升单面旗帜的有效范围，并让玩家放置前能够辨认当前声明边界，同时修复只更新底色而丢失花纹的问题。

### 2026-07-12 - 网络端口主动输入输出

**变更内容**: 为输入 / 输出端口新增三种工作模式、27 格状态与配置界面、黑白名单和六面主动 IO。

**变更理由**: 让存储网络无需额外升级组件即可主动连接相邻容器，同时保留旧有 Slimefun 货运兼容性。

**影响范围**: 网络控制器调度、端口缓冲菜单、相邻虚拟存储传输、领地交互保护、语言与存储文档。

**决策依据**: 复用抽取 / 输出升级的过滤和面配置协议，但由控制器集中调度，以守恒回退处理部分接收和满仓失败。

### 2026-07-12 - 新增 SQL 后端物品存储磁盘

**变更内容**: 新增磁盘管理器与六档磁盘，扩展虚拟库存容量策略和存储基类持久化钩子，并以 Slimefun UniversalData 保存可移动磁盘内容。

**变更理由**: 大量完整 ItemStack 数据不适合写入书本 PDC；UniversalData 能按 UUID 跨方块查询并复用 Slimefun 已配置的 SQL 后端。

**影响范围**: 存储物品注册、存储网络成员扫描、虚拟库存容量判断、磁盘 UI、配置、语言与存储文档。

**决策依据**: 翻页箱已通过 Slimefun 方块数据落入同一 SQL 基础设施；磁盘因可移动而选择 UUID 作用域的 UniversalData，而非坐标作用域 BlockData 或插件自建 SQLite。

### 2026-07-11 - 玩家文本迁移到独立 i18n

**变更内容**: 把 Kotlin 中的玩家可见中文统一迁移到 `lang/zh_CN.yml`，并移除物品文本自动写入 `config.yml` 的旧机制。

**变更理由**: 统一语言资源入口，支持后续新增语言并避免配置与文本双轨漂移。

**影响范围**: 物品与分类注册、研究、菜单、命令消息、功能交互提示和展示名称。
